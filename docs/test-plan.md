# 测试计划

## 1. 测试对象

待测对象为大学生个人助手 JavaWeb 系统，包括 Spring Boot 后端、Vue 前端和 MySQL 数据库。

## 2. 实验 1：白盒测试计划

### 2.1 测试目标

验证后端核心业务逻辑是否符合需求，包括用户认证、课程隔离、任务状态更新、学习统计、预算预警、健康记录更新、习惯打卡和首页看板聚合。

### 2.2 测试范围

- 用户注册、登录和密码校验。
- 课程查询、今日课程过滤和用户数据隔离。
- 任务状态更新、逾期任务统计。
- 学习记录按日期统计、课程占比统计。
- 健康记录同日更新逻辑。
- 消费预算使用率和预警状态。
- 习惯打卡更新、连续打卡统计。
- 首页看板聚合接口。

### 2.3 测试环境

- JDK 17+
- Maven 3.6+
- MySQL 8.0
- JUnit 5
- Spring Boot Test

### 2.4 测试工具

- JUnit 5
- AssertJ
- Spring Boot Test
- Maven Surefire

### 2.5 测试策略

采用白盒测试方法，对后端服务层进行单元测试和轻量集成测试。测试时重点关注分支条件、边界数据、异常路径和用户数据隔离。

## 3. 实验 2：黑盒测试计划

### 3.1 测试目标

从用户角度验证系统主要功能是否可用，确认页面操作、输入校验、业务流程和统计展示是否满足大学生日常管理需求。

### 3.2 测试范围

- 注册和登录。
- 首页看板展示。
- 课程、任务、日程管理。
- 学习记录和学习分析。
- 健康记录。
- 收支记录和预算预警。
- 目标创建和习惯打卡。
- 个人资料维护。

### 3.3 测试环境

- 浏览器：Microsoft Edge 或 Google Chrome
- 前端地址：`http://localhost:5173`
- 后端地址：`http://localhost:8080`
- 数据库：MySQL 8.0

### 3.4 测试工具

- 浏览器手工测试
- MySQL 客户端
- Maven
- npm/Vite

### 3.5 测试策略

采用等价类划分、边界值分析、场景法和错误推测法设计黑盒测试用例。测试过程覆盖正常流程、异常输入、重复数据和跨模块统计展示。
